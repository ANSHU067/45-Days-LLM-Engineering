Your GitHub
Is Your Resume

For builders, a link to working code beats a PDF of claims. Let's make yours say "hire me."

This is where the proof lives. The other levers point people here.
The shift

"Send me your GitHub"

Increasingly, the first thing a technical interviewer does is open your GitHub — before they read your resume.

A resume claims you can build. GitHub proves it. Empty or messy GitHub = a silent red flag.

What they check

Three things, in order

👤 Profile README

The intro that shows on your profile page — your dev homepage.

📌 Pinned repos

Your 6 best projects, front and centre. Your portfolio shelf.

📅 Activity graph

The green squares — a signal you actually code regularly.

Your dev homepage

The profile README

Create a repo named your-username/your-username with a README.md — GitHub shows it on your profile.

  • One line on what you build + your focus.
  • What you're learning now (this course + projects).
  • Links: LinkedIn, your primary platform, email.
  • Keep it short. A recruiter skims it in seconds.
Your portfolio shelf

Pinned repos = your highlight reel

You can pin 6 repos. Use them for your best, most finished work — your 4 course projects will fill most of these by Day 45.

Each pinned repo needs a real description and a good README. An impressive project with an empty README looks abandoned.

The README sells it

What a good project README has

  • What it does — one clear sentence + a screenshot or demo GIF.
  • Live link — the deployed app (everything in this course ships).
  • Tech used — Python, Gemini, Streamlit, etc.
  • How to run it — so anyone can try it in 2 minutes.

Tip: the READMEs you're already writing in this repo are exactly this skill.

About those green squares

Consistency > chasing the streak

A steady graph signals discipline — but don't fake it with junk commits. Quality of repos beats quantity of squares.

Good news: doing this course daily and committing your work naturally fills the graph.

Start now

Make this course your first pinned repo

Push your course work to a public repo with a clean README, and pin it. By Phase 1's end you'll already have a deployed project to add.

Don't wait until you're "good enough." Your learning repo is itself proof of effort.
